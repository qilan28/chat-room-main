# 在线用户显示修复方案

## 🐛 问题分析

### 原有问题

1. **只在发送消息时更新在线状态**
   - 用户登录后不发消息，就不会出现在在线列表
   - 用户只浏览不发言，会被认为离线

2. **使用用户名作为key**
   ```javascript
   onlineUsers[newMessage.username] = { ... }
   ```
   - 用户名可能重复
   - 无法准确识别用户

3. **超时时间过长**
   - 5分钟不活动才清理
   - 导致已离线用户仍显示在线

4. **没有心跳机制**
   - 无法主动保持在线状态
   - 依赖用户发送消息

## ✅ 完整解决方案

### 1. 后端改进

#### 1.1 使用用户ID作为key
```javascript
// 在线用户管理（使用userId作为key）
let onlineUsers = {}

// 更新用户在线状态
function updateUserOnlineStatus(userId, username, avatar) {
  onlineUsers[userId] = {
    id: userId,
    username: username,
    avatar: avatar,
    lastActive: Date.now()
  }
}
```

#### 1.2 缩短超时时间
```javascript
// 每2分钟清理一次不活跃用户（超过3分钟未活动）
setInterval(() => {
  const now = Date.now()
  Object.keys(onlineUsers).forEach((userId) => {
    if (now - onlineUsers[userId].lastActive > 3 * 60 * 1000) {
      console.log(`用户 ${onlineUsers[userId].username} 超时下线`)
      delete onlineUsers[userId]
    }
  })
}, 2 * 60 * 1000)
```

#### 1.3 添加心跳API
```javascript
// 心跳API - 保持在线状态
app.post('/heartbeat', (req, res) => {
  const token = req.headers.authorization?.replace('Bearer ', '')
  
  verifyUserToken(token, (isValid, session) => {
    if (!isValid) {
      return res.status(401).json({ error: '未登录' })
    }
    
    // 获取用户信息并更新在线状态
    fs.readFile(ACCOUNTS_FILE, 'utf-8', (err, data) => {
      if (err) {
        return res.status(500).json({ error: '服务器错误' })
      }
      
      try {
        const accounts = JSON.parse(data)
        const user = accounts.find(u => u.id === session.userId)
        
        if (user) {
          updateUserOnlineStatus(user.id, user.username, user.avatar)
          res.json({ success: true, onlineCount: Object.keys(onlineUsers).length })
        } else {
          res.status(404).json({ error: '用户不存在' })
        }
      } catch (parseErr) {
        res.status(500).json({ error: '服务器错误' })
      }
    })
  })
})
```

### 2. 前端改进

#### 2.1 添加心跳函数
```javascript
// 心跳机制 - 保持在线状态
async function sendHeartbeat() {
  if (!userToken) return
  
  try {
    const response = await fetch(`${SRC_URL}/heartbeat`, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${userToken}`
      }
    })
    
    if (response.ok) {
      const data = await response.json()
      // 心跳成功，更新在线用户列表
      fetchOnlineUsers()
    }
  } catch (error) {
    // 静默失败
  }
}
```

#### 2.2 登录后立即发送心跳
```javascript
async function init() {
  const isAuthenticated = await checkAuth()
  if (!isAuthenticated) return

  // ... 其他初始化代码 ...
  
  // 立即发送一次心跳，标记为在线
  sendHeartbeat()
  
  // 每30秒发送一次心跳
  setInterval(sendHeartbeat, 30000)
}
```

#### 2.3 改进在线用户渲染
```javascript
async function fetchOnlineUsers() {
  try {
    const response = await fetch(`${SRC_URL}/online-users`)
    if (!response.ok) {
      // 如果获取失败，至少显示当前用户
      if (currentUser) {
        renderOnlineUsers([currentUser])
      }
      return
    }

    const users = await response.json()
    // 确保当前用户在列表中
    if (currentUser && !users.some(u => u.id === currentUser.id)) {
      users.push(currentUser)
    }
    renderOnlineUsers(users)
  } catch (error) {
    // 静默失败，显示当前用户
    if (currentUser) {
      renderOnlineUsers([currentUser])
    }
  }
}
```

## 📊 工作流程

### 用户登录流程
```
用户登录
  ↓
前端获取token和用户信息
  ↓
调用init()初始化
  ↓
立即发送心跳 (sendHeartbeat)
  ↓
后端更新onlineUsers[userId]
  ↓
前端获取在线用户列表
  ↓
显示在侧栏
```

### 心跳保活流程
```
每30秒
  ↓
前端发送心跳请求
  ↓
后端验证token
  ↓
更新lastActive时间
  ↓
返回在线用户数量
  ↓
前端刷新在线用户列表
```

### 超时清理流程
```
每2分钟
  ↓
后端检查所有在线用户
  ↓
计算距离lastActive的时间
  ↓
超过3分钟？
  ↓ 是
删除该用户
  ↓
下次请求时列表已更新
```

### 心跳间隔: 30秒
```javascript

```
- 足够频繁保持在线
- 不会造成服务器压力
- 网络开销小

### 超时时间: 3分钟
```javascript
if (now - onlineUsers[userId].lastActive > 3 * 60 * 1000)
```
- 允许短暂的网络波动
- 快速清理真正离线的用户
- 平衡准确性和容错性

### 清理间隔: 2分钟
```javascript
setInterval(() => { ... }, 2 * 60 * 1000)
```
- 定期清理不活跃用户
- 不会太频繁影响性能
