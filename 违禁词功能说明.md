# 违禁词功能说明

## 实现时间
2026-05-06

## 功能概述
实现了完整的违禁词管理系统，包括后台管理和自动拦截功能。

## 功能特性

### 1. 后台管理功能
- ✅ 添加违禁词
- ✅ 删除违禁词
- ✅ 查看违禁词列表
- ✅ 实时统计违禁词数量

### 2. 自动拦截功能
- ✅ 发送消息时自动检测违禁词
- ✅ 包含违禁词的消息无法发送
- ✅ 提示用户消息包含违禁词
- ✅ 不区分大小写检测

## 技术实现

### 后端 API

#### 1. 获取违禁词列表
```
GET /banned-words
Authorization: Bearer {adminToken}
```

#### 2. 添加违禁词
```
POST /banned-words/add
Authorization: Bearer {adminToken}
Content-Type: application/json

{
  "word": "违禁词"
}
```

#### 3. 删除违禁词
```
POST /banned-words/delete
Authorization: Bearer {adminToken}
Content-Type: application/json

{
  "word": "违禁词"
}
```

#### 4. 批量更新违禁词
```
POST /banned-words/update
Authorization: Bearer {adminToken}
Content-Type: application/json

{
  "words": ["违禁词1", "违禁词2"]
}
```

## 使用说明

### 管理员操作步骤

1. **登录后台**
   - 访问 http://localhost:3000/admin-login.html
   - 使用管理员账号登录

2. **打开违禁词管理**
   - 进入消息管理系统
   - 点击"违禁词管理"按钮

3. **添加违禁词**
   - 在输入框中输入要禁止的词语
   - 点击"添加"按钮或按回车键
   - 系统会提示添加成功

4. **删除违禁词**
   - 找到要删除的违禁词
   - 点击右侧的删除按钮
   - 确认删除操作

### 用户体验

当用户发送包含违禁词的消息时：
1. 消息无法发送
2. 收到错误提示："消息包含违禁词，无法发送"
3. 可以看到具体包含哪些违禁词（开发模式下）

## 安全特性

1. **权限控制**
   - 只有管理员可以管理违禁词
   - 使用 Token 验证身份

2. **XSS 防护**
   - 前端显示时进行 HTML 转义
   - 防止恶意脚本注入

3. **实时生效**
   - 添加/删除违禁词立即生效
   - 无需重启服务器

## 注意事项

1. **检测规则**
   - 不区分大小写
   - 只要消息中包含违禁词就会被拦截
   - 支持中文、英文等多语言

2. **性能考虑**
   - 违禁词检测使用同步读取
   - 建议违禁词数量控制在合理范围
   - 如需大量违禁词，建议优化为缓存机制

3. **管理建议**
   - 定期审查违禁词列表
   - 避免添加过于宽泛的词语
   - 考虑用户体验，合理设置违禁词

## 故障排除

### 问题1：违禁词不生效
- 检查 `data/banned_words.json` 文件是否存在
- 检查文件格式是否正确
- 重启服务器

### 问题2：无法添加违禁词
- 检查是否已登录管理员账号
- 检查 Token 是否有效
- 查看浏览器控制台错误信息

### 问题3：违禁词重复
- 系统会自动检查重复
- 如果提示已存在，说明该词已在列表中
