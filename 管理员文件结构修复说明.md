# 管理员文件结构修复说明

## 修复时间
2026-05-05

## 问题描述
服务器启动时出现错误：
```
解析管理员文件失败: TypeError: Cannot read properties of undefined (reading 'find')
```

## 问题原因

### 1. admin.json 文件结构错误
**错误的结构：**
```json
{
  "username": "admin",
  "password": "admin123",
  "sessions": [...]
}
```

**正确的结构：**
```json
{
  "admins": [
    {
      "id": "admin001",
      "username": "admin",
      "password": "admin123",
      "role": "super",
      "createdAt": "2025-01-01T00:00:00.000Z"
    }
  ],
  "sessions": []
}
```

代码期望 `adminData.admins` 是一个数组，但旧文件中没有 `admins` 字段，导致 `undefined.find()` 错误。

### 2. heartbeat 接口代码错误
在 `index.js` 第344行，代码错误地使用了：
```javascript
const user = accounts.find(u => u.id === session.userId)
```

应该是：
```javascript
const user = accountsData.users.find(u => u.id === session.userId)
```

因为 `accounts.json` 的结构是：
```json
{
  "users": [...],
  "sessions": {...}
}
```

## 修复内容

### 1. 修复 admin.json 文件
- 将单个管理员对象改为 `admins` 数组结构
- 添加必要的字段：`id`, `role`, `createdAt`
- 清空 `sessions` 数组（旧的会话已过期）

### 2. 修复 index.js 代码
- 修改第343-344行，将 `accounts` 改为 `accountsData.users`
- 确保正确访问用户数组

## 默认管理员账号
- **用户名**: admin
- **密码**: admin123
- **角色**: super（超级管理员）

## 注意事项
- 如果将来需要重置管理员账号，请确保使用正确的 JSON 结构
- `admin.json` 必须包含 `admins` 数组和 `sessions` 数组
- 所有管理员必须有唯一的 `id` 字段
