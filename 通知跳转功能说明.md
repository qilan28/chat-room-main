# 通知跳转功能实现说明

## ✅ 功能描述

点击通知后：
1. ✅ 自动跳转到对应的消息位置
2. ✅ 消息会高亮显示（已有功能）
3. ✅ 标记通知为已读
4. ✅ 关闭通知面板
5. ✅ 更新未读徽章数量

## 🔧 实现逻辑

### 点击通知时的处理流程

```javascript
item.addEventListener('click', () => {
  // 1. 标记为已读
  markNotificationAsRead(notif.id)
  
  // 2. 关闭通知面板
  document.getElementById('notification-panel').classList.remove('active')
  
  // 3. 跳转到对应消息
  if (notif.data.messageId) {
    scrollToMessage(notif.data.messageId)
  }
})
```

### scrollToMessage 函数

```javascript
function scrollToMessage(messageId) {
  const targetMessage = document.querySelector(`[data-message-id="${messageId}"]`)
  if (targetMessage) {
    // 平滑滚动到消息位置，居中显示
    targetMessage.scrollIntoView({ behavior: 'smooth', block: 'center' })
    
    // 高亮显示消息（3秒后消失）
    targetMessage.classList.add('highlight')
    setTimeout(() => {
      targetMessage.classList.remove('highlight')
    }, 3000)
  }
}
```

### markNotificationAsRead 函数

```javascript
async function markNotificationAsRead(notificationId) {
  try {
    await fetch(`${SRC_URL}/notifications/read`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${userToken}`,
      },
      body: JSON.stringify({ notificationIds: [notificationId] }),
    })

    // 更新本地状态
    const notif = notifications.find((n) => n.id === notificationId)
    if (notif) {
      notif.read = true
      updateNotificationBadge()  // 更新徽章
      renderNotifications()       // 重新渲染列表
    }
  } catch (error) {
    console.error('标记通知失败:', error)
  }
}
```

## 🔍 错误处理

### 消息不存在
如果消息已被删除或不在当前加载的消息列表中：
- `scrollToMessage` 会静默失败（不会报错）
- 通知仍会被标记为已读
- 通知面板仍会关闭

### 网络错误
如果标记已读的API调用失败：
- 通知仍会跳转到消息
- 通知面板仍会关闭
- 但通知不会被标记为已读（下次打开仍显示为未读）
