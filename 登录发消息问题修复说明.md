# 登录后无法发消息问题修复说明

## 🐛 问题描述

登录后无法发送消息，消息发送功能不工作。

## 🔍 问题原因

1. **SRC_URL和WS_URL定义错误**
   - 原代码中 `SRC_URL` 和 `WS_URL` 被定义为空字符串
   - 导致API请求地址错误

2. **用户信息使用错误**
   - 发送消息时使用的是旧的 `userConfig.username` 和 `userConfig.avatar`
   - 登录后应该使用 `currentUser` 的信息

3. **消息归属判断错误**
   - 在 `fetchMessages` 函数中判断是否是自己的消息时，使用的是 `userConfig.username`
   - 应该使用登录后的 `currentUser.username`

## ✅ 修复内容

### 1. 修复SRC_URL和WS_URL定义

**修改前**:
```javascript
const SRC_URL = '' // 留空表示同
const WS_URL = '' // 留空表示同
```

**修改后**:
```javascript
const SRC_URL = window.location.origin // 服务器地址
const WS_URL = window.location.origin.replace('http', 'ws') // WebSocket地址
```

### 2. 修复发送消息时的用户信息

**修改前**:
```javascript
const message = {
  username: userConfig.username,
  avatar: userConfig.avatar,
  text: text,
  // ...
}
```

**修改后**:
```javascript
// 使用登录用户信息
const username = currentUser ? currentUser.username : (userConfig.username || '用户')
const avatar = currentUser ? currentUser.avatar : (userConfig.avatar || 'https://ui-avatars.com/api/?name=User&background=random')

const message = {
  username: username,
  avatar: avatar,
  text: text,
  // ...
}
```

### 3. 修复消息归属判断

**修改前**:
```javascript
addedMessages.forEach((message) => {
  const isSelf = message.username === userConfig.username
  displayMessage(message, isSelf)
})
```

**修改后**:
```javascript
addedMessages.forEach((message) => {
  // 使用登录用户信息判断
  const currentUsername = currentUser ? currentUser.username : userConfig.username
  const isSelf = message.username === currentUsername
  displayMessage(message, isSelf)
})
```

### 4. 移除重复的变量定义

**修改前**:
```javascript
function setupWebSocket() {
  // ...
  const SRC_URL = window.location.origin  // 重复定义
  const WS_URL = window.location.origin.replace('http', 'ws')  // 重复定义
  // ...
}
```

**修改后**:
```javascript
function setupWebSocket() {
  // 直接使用全局的 SRC_URL 和 WS_URL
  const wsurl = WS_URL ? WS_URL : window.location.host
  // ...
}
```
