# 聊天室新功能实现说明

## 🎯 新增功能概览

### 1. ✅ 用户注册登录系统
- 用户注册（用户名、密码、邮箱）
- 用户登录（生成7天有效token）
- 会话管理
- 登录后才能使用聊天室

### 2. ✅ 私聊功能
- 一对一私聊
- 私聊消息列表
- 私聊会话管理
- 未读消息提示

### 3. ✅ @提及功能
- 在消息中@其他用户
- @提示和高亮显示
- 点击@跳转到用户资料

### 4. ✅ 离线消息通知系统
- 被@时创建通知
- 被回复时创建通知
- 收到私聊时创建通知
- 离线也能收到通知（下次登录查看）

## 📋 后端API接口

### 用户认证

```javascript
// 注册
POST /auth/register
Body: { username, password, email }
Response: { success, userId, username }

// 登录
POST /auth/login
Body: { username, password }
Response: { success, token, user: { id, username, avatar, email } }

// 获取当前用户
GET /auth/me
Headers: Authorization: Bearer {token}
Response: { id, username, avatar, email }

// 登出
POST /auth/logout
Headers: Authorization: Bearer {token}
Response: { success }
```

### 用户列表

```javascript
// 获取所有用户（用于@和私聊）
GET /users/list
Headers: Authorization: Bearer {token}
Response: { users: [{ id, username, avatar }] }
```

### 私聊功能

```javascript
// 发送私聊消息
POST /private/send
Headers: Authorization: Bearer {token}
Body: { toUserId, text, attachments }
Response: { success, message }

// 获取与某用户的私聊记录
GET /private/messages/:userId
Headers: Authorization: Bearer {token}
Response: { messages: [...] }

// 获取私聊会话列表
GET /private/conversations
Headers: Authorization: Bearer {token}
Response: { conversations: [{ userId, username, lastMessage, unreadCount }] }
```

### 通知系统

```javascript
// 获取通知列表
GET /notifications
Headers: Authorization: Bearer {token}
Response: { notifications: [...] }

// 标记通知为已读
POST /notifications/read
Headers: Authorization: Bearer {token}
Body: { notificationIds: [...] }
Response: { success }
```

1. **密码安全**

2. **Token管理**
   - Token有效期7天
   - 需要定期清理过期session

## 📊 数据流程

### 注册流程
```
用户填写表单 → 前端验证 → POST /auth/register → 
检查用户名 → 创建用户 → 返回成功
```

### 登录流程
```
用户输入账号密码 → POST /auth/login → 
验证账号密码 → 生成token → 存储到localStorage → 
跳转到聊天室
```

### 私聊流程
```
选择用户 → 输入消息 → POST /private/send → 
保存消息 → 创建通知 → WebSocket推送给对方 → 
对方收到消息
```

### @通知流程
```
输入@用户名 → 发送消息 → 检测@内容 → 
创建mention通知 → 保存到notifications.json → 
对方登录时查看通知
```

### 回复通知流程
```
引用消息 → 发送回复 → 检测引用 → 
创建reply通知 → 保存通知 → 
被回复用户收到通知
```
