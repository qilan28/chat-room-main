# 消息显示问题调试指南

## 🔍 问题现象

- 登录后界面显示正常
- 用户信息显示正常（用户名、头像）
- 但是消息区域是空的，无法发送和显示消息

## 🧪 调试步骤

### 1. 打开浏览器控制台（F12）

查看是否有以下错误：
- ❌ API请求失败
- ❌ JavaScript错误
- ❌ 网络错误

### 2. 检查网络请求

在控制台的 **Network（网络）** 标签中：

1. **检查 `/messages` 请求**
   - 应该返回 200 状态码
   - 响应应该是消息数组

2. **检查 `/online-users` 请求**
   - 应该返回 200 状态码
   - 响应应该是用户列表

3. **检查 `/config` 请求**
   - 应该返回 200 状态码
   - 响应应该是配置对象

### 3. 在控制台执行测试命令

```javascript
// 1. 检查变量是否正确定义
console.log('userToken:', localStorage.getItem('userToken'))
console.log('currentUser:', JSON.parse(localStorage.getItem('currentUser')))
console.log('SRC_URL:', window.location.origin)

// 2. 测试获取消息
fetch(window.location.origin + '/messages')
  .then(r => r.json())
  .then(data => console.log('消息列表:', data))
  .catch(e => console.error('获取消息失败:', e))

// 3. 测试发送消息
fetch(window.location.origin + '/messages', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: '测试用户',
    avatar: 'https://ui-avatars.com/api/?name=Test',
    text: '测试消息',
    attachments: [],
    timestamp: new Date().toISOString(),
    id: 'test' + Date.now()
  })
})
  .then(r => r.json())
  .then(data => console.log('发送成功:', data))
  .catch(e => console.error('发送失败:', e))

// 4. 检查消息容器
console.log('消息容器:', document.getElementById('messages-container'))
console.log('消息数量:', document.querySelectorAll('.message').length)
```

## 🔧 可能的问题和解决方案

### 问题1: localStorage数据冲突

**症状**: 旧的用户配置干扰新的登录系统

**解决方案**:
```javascript
// 在控制台执行，清除所有数据
localStorage.clear()
// 然后刷新页面重新登录
```

### 问题2: 消息数据格式错误

**症状**: 消息文件损坏或格式不正确

**解决方案**:
1. 停止服务器
2. 删除或重命名 `data/messages.json`
3. 重启服务器（会自动创建新文件）

### 问题3: WebSocket连接失败

**症状**: 无法接收实时消息

**解决方案**:
```javascript
// 在控制台检查WebSocket状态
// 应该看到 "WebSocket连接已建立" 的日志
```

### 问题4: 用户未正确登录

**症状**: token无效或过期

**解决方案**:
```javascript
// 检查token
fetch(window.location.origin + '/auth/me', {
  headers: { 'Authorization': 'Bearer ' + localStorage.getItem('userToken') }
})
  .then(r => r.json())
  .then(data => console.log('当前用户:', data))
  .catch(e => console.error('验证失败:', e))
```

## 🐛 常见错误信息

### "userToken is not defined"
- **原因**: 变量未定义
- **状态**: ✅ 已修复

### "Failed to fetch"
- **原因**: 服务器未运行或URL错误
- **解决**: 检查服务器是否在运行

### "401 Unauthorized"
- **原因**: 未登录或token过期
- **解决**: 重新登录

### "Cannot read property 'username' of null"
- **原因**: currentUser为null
- **解决**: 检查登录状态
