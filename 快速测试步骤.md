# 快速测试步骤

## 🚀 现在请按以下步骤测试

### 步骤1: 清除缓存（重要！）
```javascript
// 在浏览器控制台（F12）执行：
localStorage.clear()
```

### 步骤2: 刷新页面
按 `F5` 或 `Ctrl+R` 刷新浏览器

### 步骤3: 查看控制台日志
打开控制台（F12），应该看到：
```
=== 聊天室初始化开始 ===
userToken: null (或你的token)
currentUser: null (或你的用户信息)
SRC_URL: http://localhost:3000
```

### 步骤4: 登录
- 如果跳转到登录页，输入账号密码登录
- 如果已经登录，跳过此步

### 步骤5: 查看初始化日志
登录后，控制台应该显示：
```
用户已登录，开始加载数据
开始获取消息...
消息API响应状态: 200
获取到消息数量: X
新增消息数量: X
=== 聊天室初始化完成 ===
```

### 步骤6: 测试发送消息
1. 在输入框输入 "测试消息123"
2. 按回车或点击发送按钮
3. 查看控制台日志：
```
准备发送消息: 测试消息123
使用登录用户信息
```

### 步骤7: 检查消息显示
- 消息应该出现在聊天区域
- 自己的消息在右侧（蓝色）
- 控制台应该显示：
```
渲染消息: 测试消息123 isSelf: true
```

## 🔧 常见问题快速修复

### 问题1: "userToken: null"
**解决**: 需要重新登录
```javascript
localStorage.clear()
// 刷新页面，重新登录
```

### 问题2: "消息API响应状态: 404"
**解决**: 服务器未正确启动
```bash
# 重启服务器
npm start
```

### 问题3: "新增消息数量: 0"（但实际有消息）
**解决**: localStorage中的旧数据干扰
```javascript
localStorage.removeItem('message_data')
// 刷新页面
```

### 问题4: 消息发送后没有显示
**解决**: 检查是否有错误日志，可能是：
- WebSocket未连接
- 消息格式错误
- 用户信息缺失

## 💡 调试技巧

### 手动发送测试消息
```javascript
// 在控制台执行
fetch('http://localhost:3000/messages', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: '测试',
    avatar: 'https://ui-avatars.com/api/?name=Test',
    text: '这是一条测试消息',
    attachments: [],
    timestamp: new Date().toISOString(),
    id: 'test' + Date.now()
  })
})
.then(r => r.json())
.then(d => console.log('发送成功:', d))
.catch(e => console.error('发送失败:', e))
```

### 检查消息容器
```javascript
// 在控制台执行
const container = document.getElementById('messages-container')
console.log('消息容器存在:', !!container)
console.log('当前消息数:', container.querySelectorAll('.message').length)
```

### 强制刷新消息
```javascript
// 在控制台执行
fetchMessages()
```
