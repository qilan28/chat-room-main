# userToken未定义问题修复

## 🐛 错误信息
```
script.js:921 Uncaught (in promise) ReferenceError: userToken is not defined
    at checkAuth (script.js:921:3)
    at init (script.js:996:33)
```

## 🔍 问题原因

`userToken` 和 `currentUser` 等认证相关变量没有在文件开头正确定义，导致在 `checkAuth()` 函数中使用时报错"未定义"。

## ✅ 修复方案

在 `script.js` 文件开头（第11-16行）添加认证相关变量的定义：

```javascript
// 用户认证变量
let userToken = localStorage.getItem('userToken')
let currentUser = JSON.parse(localStorage.getItem('currentUser') || 'null')
let allUsers = []
let notifications = []
let privateConversations = []
```

## 📝 完整修改位置

**文件**: `public/src/script.js`

**位置**: 第7-16行

**修改后的代码**:
```javascript
const SRC_URL = window.location.origin // 服务器地址
const WS_URL = window.location.origin.replace('http', 'ws') // WebSocket地址
const electron_build = false // 是否为electron构建

let userToken = localStorage.getItem('userToken')
```

## 💡 为什么会出现这个问题

之前在添加登录功能时，在代码中间位置添加了这些变量的使用，但忘记在文件开头声明它们。JavaScript 的变量作用域要求变量必须先声明后使用。

## 📋 变量说明

- `userToken`: 用户登录后的认证令牌（7天有效）
- `currentUser`: 当前登录用户的信息对象
- `allUsers`: 所有用户列表（用于@功能）
- `notifications`: 通知列表
- `privateConversations`: 私聊会话列表
