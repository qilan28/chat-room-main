## 技术实现

### 后端 API

#### 1. 获取用户列表
```
GET /admin/users
Authorization: Bearer {adminToken}

Response:
{
  "users": [
    {
      "id": "user1777911944170",
      "username": "qilan",
      "email": "3100676552@qq.com",
      "avatar": "https://ui-avatars.com/api/?name=qilan&background=random",
      "createdAt": "2026-05-04T16:25:44.170Z",
      "lastLoginAt": "2026-05-04T16:25:56.192Z"
    }
  ]
}
```

#### 2. 编辑用户信息
```
PUT /admin/users/:id
Authorization: Bearer {adminToken}
Content-Type: application/json

{
  "username": "新用户名",
  "email": "new@email.com",
  "password": "新密码" // 可选
}

Response:
{
  "success": true,
  "user": {
    "id": "user123",
    "username": "新用户名",
    "email": "new@email.com",
    "avatar": "..."
  }
}
```

#### 3. 删除用户
```
DELETE /admin/users/:id
Authorization: Bearer {adminToken}

Response:
{
  "success": true
}
```

#### 4. 重置用户密码
```
POST /admin/users/:id/reset-password
Authorization: Bearer {adminToken}
Content-Type: application/json

{
  "newPassword": "新密码"
}

Response:
{
  "success": true
}
```

### 前端界面

#### 用户列表界面
- 卡片式布局，美观易用
- 显示用户头像和详细信息
- 每个用户卡片有编辑和删除按钮
- 顶部有搜索框和用户数量统计

#### 编辑用户界面
- 模态框形式
- 表单包含：用户ID（只读）、用户名、邮箱、新密码
- 密码可选填，留空则不修改
- 保存和取消按钮

### 权限控制
- 所有接口都需要管理员 token 验证
- 使用 `verifyAdminToken()` 函数验证身份
- 未授权访问返回 403 错误

## 使用说明

### 管理员操作步骤

#### 1. 查看用户列表
1. 登录管理后台
2. 点击"用户管理"按钮
3. 查看所有注册用户

#### 2. 搜索用户
1. 在用户管理界面
2. 在搜索框输入关键词
3. 自动过滤显示匹配的用户

#### 3. 编辑用户
1. 点击用户卡片右侧的"编辑"按钮
2. 修改用户名、邮箱或密码
3. 点击"保存"按钮
4. 系统提示更新成功

#### 4. 删除用户
1. 点击用户卡片右侧的"删除"按钮
2. 确认删除操作
3. 用户账号和会话被删除

## 数据处理

### 编辑用户时
1. **修改用户名**
   - 检查用户名是否重复
   - 自动更新头像URL
   - 保存到 accounts.json

2. **修改邮箱**
   - 直接更新邮箱字段
   - 可以设置为空

3. **修改密码**
   - 验证密码长度（至少6位）
   - 只有填写时才更新
   - 留空则保持原密码

### 删除用户时
1. 从 users 数组中移除用户
2. 删除该用户的所有 sessions
3. 保存更新后的 accounts.json
4. 用户无法再登录

## 安全特性

### 1. 权限验证
- 所有接口都需要管理员 token
- 使用 `verifyAdminToken()` 验证
- 未授权返回 403 错误

### 2. 数据验证
- 用户名不能为空
- 密码长度至少6位
- 用户名不能重复
- 用户ID必须存在

### 3. 操作确认
- 删除用户需要二次确认
- 显示详细的操作后果
- 防止误操作

### 4. 会话管理
- 删除用户时清除所有会话
- 被删除的用户立即无法登录
- 保证系统安全性

## 注意事项

### 1. 密码管理
⚠️ **当前密码是明文存储**
- 生产环境应使用 bcrypt 等加密
- 建议实现密码哈希存储
- 增强安全性

### 2. 用户删除
⚠️ **删除操作不可恢复**
- 删除前请确认
- 用户的所有数据将丢失
- 建议实现软删除机制

### 3. 用户名修改
⚠️ **修改用户名的影响**
- 历史消息中的用户名不会更新
- 可能影响@提及功能
- 建议谨慎修改

### 4. 搜索功能
- 搜索是前端过滤
- 大量用户时可能较慢
- 建议后端实现分页

## 故障排除

### 问题1：无法获取用户列表
**解决方案**:
1. 检查是否已登录管理员账号
2. 检查 token 是否有效
3. 查看浏览器控制台错误
4. 检查 accounts.json 文件格式

### 问题2：无法编辑用户
**解决方案**:
1. 检查用户名是否重复
2. 检查密码长度是否足够
3. 查看服务器日志
4. 确认 accounts.json 文件权限

### 问题3：删除用户失败
**解决方案**:
1. 检查用户ID是否正确
2. 确认有管理员权限
3. 查看服务器错误日志
4. 检查文件写入权限

### 问题4：搜索不工作
**解决方案**:
1. 刷新页面重新加载用户列表
2. 检查浏览器控制台错误
3. 确认搜索框事件绑定正确
