# 登录循环问题修复说明

## 修复时间
2026-05-06

## 问题描述
在登录页面会出现无限循环跳转的问题，表现为：
1. 用户访问 `/admin-login.html`
2. 页面检测到 localStorage 中有 token
3. 显示"检测到已登录，正在跳转..."
4. 跳转到 `/message_manage?token=xxx`
5. 后端验证 token 失败（可能已过期）
6. 重定向回 `/admin-login.html`
7. 回到步骤 2，形成无限循环

## 问题原因

### 根本原因
登录页面只检查 localStorage 中是否存在 token，但不验证 token 是否有效。当 token 过期或无效时：
- localStorage 中仍然保存着旧的 token
- 页面检测到 token 存在就尝试跳转
- 后端验证失败后重定向回登录页
- 登录页再次检测到 token，再次跳转
- 形成死循环

### 触发条件
1. Token 已过期（超过 24 小时）
2. Token 被手动删除或失效
3. admin.json 文件被重置
4. Sessions 数据被清空

## 修复方案

### 1. 添加 Token 验证接口

**文件**: `index.js`

添加了一个新的 API 接口用于验证 token：

```javascript
// 验证管理员token接口
app.get('/admin/verify', (req, res) => {
  const token = req.headers.authorization?.replace('Bearer ', '')

  if (!token) {
    return res.json({ valid: false })
  }

  verifyAdminToken(token, (isValid, session) => {
    if (isValid) {
      res.json({ 
        valid: true, 
        username: session.username, 
        role: session.role 
      })
    } else {
      res.json({ valid: false })
    }
  })
})
```

**功能**:
- 接收 token 并验证其有效性
- 返回验证结果和用户信息
- 不会重定向，只返回 JSON 数据

### 2. 修改登录页面逻辑

**文件**: `public/admin-login.html`

**修改前**:
```javascript
const adminToken = localStorage.getItem('adminToken')
if (adminToken) {
  toastr.info('检测到已登录，正在跳转...')
  setTimeout(() => {
    window.location.href = `/message_manage?token=${adminToken}`
  }, 1000)
}
```

**修改后**:
```javascript
const adminToken = localStorage.getItem('adminToken')
if (adminToken) {
  // 验证 token 是否有效
  fetch('/admin/verify', {
    method: 'GET',
    headers: {
      Authorization: `Bearer ${adminToken}`,
    },
  })
    .then((response) => response.json())
    .then((data) => {
      if (data.valid) {
        toastr.info('检测到已登录，正在跳转...')
        setTimeout(() => {
          window.location.href = `/message_manage?token=${adminToken}`
        }, 1000)
      } else {
        // Token 无效，清除本地存储
        localStorage.removeItem('adminToken')
        localStorage.removeItem('adminUsername')
      }
    })
    .catch((error) => {
      console.error('验证 token 失败:', error)
      // 验证失败，清除本地存储
      localStorage.removeItem('adminToken')
      localStorage.removeItem('adminUsername')
    })
}
```

**改进点**:
1. 在跳转前先调用 `/admin/verify` 验证 token
2. 只有 token 有效时才跳转
3. Token 无效时清除 localStorage
4. 网络错误时也清除 localStorage

## 修复效果

### 修复前
```
访问登录页 → 检测到 token → 跳转后台 → 验证失败 → 重定向登录页 → 循环
```

### 修复后
```
访问登录页 → 检测到 token → 验证 token
    ↓
有效 → 跳转后台
    ↓
无效 → 清除 localStorage → 停留在登录页
```

## 技术细节

### Token 验证流程
```javascript
1. 前端发送请求到 /admin/verify
2. 携带 Authorization: Bearer {token}
3. 后端调用 verifyAdminToken() 函数
4. 检查 token 是否存在于 sessions 中
5. 检查 token 是否过期
6. 返回验证结果 { valid: true/false }
```

### 清除策略
- Token 无效 → 清除 localStorage
- 验证失败 → 清除 localStorage
- 网络错误 → 清除 localStorage
- 确保不会保留无效的登录状态
