# 在线用户和滚动位置修复

## ✅ 问题1: 在线用户显示不稳定

### 问题描述
- 在线用户列表时有时无
- 有时显示"在线用户"标题但下面是空的
- 刷新后可能看不到任何用户

### 原因分析
1. API请求失败时没有降级处理
2. 当前用户可能不在在线用户列表中
3. 错误处理会显示toastr提示，影响用户体验

### 修复方案

#### 1. 添加降级处理
```javascript
async function fetchOnlineUsers() {
  try {
    const response = await fetch(`${SRC_URL}/online-users`)
    if (!response.ok) {
      // 如果获取失败，至少显示当前用户
      if (currentUser) {
        renderOnlineUsers([currentUser])
      }
      return
    }
    
    const users = await response.json()
    // 确保当前用户在列表中
    if (currentUser && !users.some(u => u.id === currentUser.id)) {
      users.push(currentUser)
    }
    renderOnlineUsers(users)
  } catch (error) {
    // 静默失败，显示当前用户
    if (currentUser) {
      renderOnlineUsers([currentUser])
    }
  }
}
```

#### 2. 确保当前用户始终显示
- 即使API失败，也会显示当前登录用户
- 如果API返回的列表中没有当前用户，会自动添加
- 不再显示错误提示，避免干扰用户

### 修复效果
- ✅ 在线用户列表始终至少显示当前用户
- ✅ 不会出现空列表的情况
- ✅ 网络错误时静默处理，不影响使用
- ✅ 刷新页面后用户列表稳定显示

## ✅ 问题2: 默认滚动位置在顶部

### 问题描述
- 打开聊天室时，消息容器默认在顶部
- 需要手动滚动才能看到最新消息
- 刷新页面后又回到顶部

### 期望行为
- 打开聊天室时自动滚动到底部
- 刷新页面后仍然在底部
- 可以看到最新的消息

### 修复方案

#### 1. 初始化本地消息后滚动
```javascript
function initLocalMessages() {
  // ... 加载本地消息 ...
  
  // 初始化后滚动到底部
  setTimeout(() => {
    messagesContainer.scrollTop = messagesContainer.scrollHeight
  }, 100)
}
```

#### 2. 首次获取消息后滚动
```javascript
async function fetchMessages() {
  // ... 获取并显示消息 ...
  
  // 如果是首次加载，滚动到底部
  if (addedMessages.length > 0 && lattttMessages.length === addedMessages.length) {
    setTimeout(() => {
      messagesContainer.scrollTop = messagesContainer.scrollHeight
    }, 100)
  }
}
```

### 滚动逻辑说明

#### 何时自动滚动到底部
1. ✅ 页面首次加载时
2. ✅ 刷新页面后
3. ✅ 发送消息后
4. ✅ 在底部时接收到新消息

#### 何时不自动滚动
1. ❌ 用户正在查看历史消息（不在底部）
2. ❌ 接收到新消息但用户在上划查看

### 修复效果
- ✅ 打开聊天室立即看到最新消息
- ✅ 刷新页面后保持在底部
- ✅ 不会打断用户查看历史消息
- ✅ 发送消息后自动滚动到底部

1. **API返回**: `/online-users` 接口
2. **降级数据**: 当前登录用户 `currentUser`
3. **去重处理**: 按用户名去重

### 显示规则
```javascript

const users = await fetch('/online-users')

// 2. 确保当前用户在列表中
if (!users.includes(currentUser)) {

}

// 3. 去重
const uniqueUsers = removeDuplicates(users)
```
