# 聊天室新功能说明

## 🎉 新增功能概览

### 1. 多管理员账号系统
- ✅ 支持多个管理员账号
- ✅ 区分超级管理员和普通管理员权限
- ✅ 超级管理员可以添加/删除其他管理员
- ✅ 每个管理员独立的会话管理

### 2. 消息右键菜单（类似QQ）
- ✅ 右键点击消息显示操作菜单
- ✅ 移动端支持长按显示菜单
- ✅ 功能包括：
  - 📋 复制消息
  - 📤 转发消息
  - 💬 引用回复
  - ↩️ 撤回消息（自己的消息）
  - 🗑️ 删除消息（管理员权限）

### 3. 用户铭牌/徽章系统
- ✅ 管理员可创建自定义铭牌
- ✅ 为指定用户分配铭牌
- ✅ 铭牌支持自定义图标、颜色、背景色
- ✅ 在聊天界面显示用户铭牌

## 📋 详细功能说明

### 一、多管理员系统

#### 管理员角色
- **超级管理员** (super): 拥有所有权限，可以管理其他管理员
- **普通管理员** (normal): 可以管理消息、公告等，但不能管理其他管理员

#### 管理员管理界面
访问路径：管理面板 → 管理员管理

功能：
1. **查看所有管理员**
   - 显示用户名、角色、创建时间
   
2. **添加管理员**
   - 输入用户名和密码
   - 选择角色（普通/超级）
   - 仅超级管理员可操作

3. **删除管理员**
   - 点击删除按钮移除管理员
   - 不能删除自己
   - 仅超级管理员可操作

#### 数据结构
```json
{
  "admins": [
    {
      "id": "admin001",
      "username": "admin",
      "password": "admin123",
      "role": "super",
      "createdAt": "2025-01-01T00:00:00.000Z"
    }
  ],
  "sessions": []
}
```

### 二、消息右键菜单

#### 使用方法
- **PC端**: 右键点击消息
- **移动端**: 长按消息（500ms）

#### 菜单选项

1. **复制** 📋
   - 复制消息文本到剪贴板
   - 所有用户可用

2. **转发** 📤
   - 将消息内容填入输入框
   - 格式：`[转发] 用户名: 消息内容`
   - 所有用户可用

3. **引用** 💬
   - 引用消息进行回复
   - 格式：`回复 @用户名: 消息内容`
   - 所有用户可用

4. **撤回** ↩️
   - 撤回自己发送的消息
   - 仅对自己的消息可见
   - 需要确认操作

5. **删除** 🗑️
   - 删除任意用户的消息
   - 仅管理员可见
   - 需要确认操作

#### 技术实现
- 右键菜单使用固定定位
- 自动防止超出屏幕边界
- 点击其他地方自动关闭
- 支持触摸事件

### 三、用户铭牌系统

#### 铭牌管理界面
访问路径：管理面板 → 铭牌管理

#### 功能说明

1. **创建铭牌**
   - 铭牌ID（唯一标识）
   - 铭牌名称
   - 图标（Emoji）
   - 文字颜色
   - 背景颜色

2. **分配铭牌**
   - 输入用户名
   - 选择要分配的铭牌（可多选）
   - 保存后立即生效

3. **预设铭牌**
   - VIP铭牌：👑 金色
   - 管理员铭牌：🛡️ 红色

#### 铭牌显示
- 显示在用户名旁边
- 支持多个铭牌同时显示
- 自动应用颜色样式

#### 数据结构
```json
{
  "badges": [
    {
      "id": "vip",
      "name": "VIP",
      "icon": "👑",
      "color": "#FFD700",
      "bgColor": "#FFF8DC"
    },
    {
      "id": "admin",
      "name": "管理员",
      "icon": "🛡️",
      "color": "#FF0000",
      "bgColor": "#FFE4E1"
    }
  ],
  "userBadges": {
    "用户名1": ["vip", "admin"],
    "用户名2": ["vip"]
  }
}
```

## 🔧 API 接口

### 管理员管理

```javascript
// 获取所有管理员（仅超级管理员）
GET /admin/list
Headers: Authorization: Bearer {token}

// 添加管理员（仅超级管理员）
POST /admin/add
Headers: Authorization: Bearer {token}
Body: {
  "username": "newadmin",
  "password": "password123",
  "role": "normal"
}

// 删除管理员（仅超级管理员）
DELETE /admin/:id
Headers: Authorization: Bearer {token}
```

### 铭牌管理

```javascript
// 获取铭牌配置
GET /badges

// 更新铭牌配置（需要管理员权限）
POST /badges
Headers: Authorization: Bearer {token}
Body: {
  "badges": [...],
  "userBadges": {...}
}

// 设置用户铭牌（需要管理员权限）
POST /badges/user
Headers: Authorization: Bearer {token}
Body: {
  "username": "用户名",
  "badgeIds": ["vip", "admin"]
}
```

## 💡 使用建议

### 管理员管理
1. 首次使用请立即修改默认管理员密码
2. 谨慎分配超级管理员权限
3. 定期检查管理员列表
4. 删除不再需要的管理员账号

### 铭牌使用
1. 为VIP用户分配VIP铭牌
2. 为管理员分配管理员铭牌
3. 可创建自定义铭牌（如：活跃用户、新人等）
4. 建议铭牌数量不超过3个/用户

### 右键菜单
1. 教育用户如何使用右键菜单
2. 移动端用户使用长按操作
3. 管理员可快速删除不当消息
4. 用户可撤回误发消息

## ⚠️ 注意事项

1. **权限控制**
   - 只有超级管理员能管理其他管理员
   - 普通管理员只能管理消息和铭牌
   - 用户只能撤回自己的消息

2. **数据安全**
   - 定期备份 `data/` 目录
   - 管理员密码使用强密码
   - 不要泄露管理员token

3. **性能优化**
   - 铭牌数据在页面加载时获取
   - 右键菜单按需显示
   - 管理员列表仅在管理面板加载
